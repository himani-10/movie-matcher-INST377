Why we built it

Movie Matcher

Picking a movie in a group is slow and often unfair. Movie Matcher collects everyone's preferences, blends them into a shared filter set, and returns a shortlist with trailers, cast, ratings, and where to watch.

How it works

1. Create or join a room

Generate a room code or join an existing one so every participant can submit preferences.

2. Capture preferences

Genre, language, runtime, and rating inputs are saved to the backend for aggregation per room.

3. Blend and match

Call TMDb discovery APIs using the combined filters to surface fair, consensus-ready results.

4. Show what to watch

Pull Watchmode streaming sources so every suggested title lists where to hit play.

APIs & data

TMDb

Discover movies by genre and language, then fetch details including runtime, vote average, cast, and trailers.

Endpoints: Discover, Movie Details, Videos, Credits.

Watchmode

Find streaming sources per title so groups only see suggestions that are playable on their platforms.

Endpoints: Title search + streaming sources.

Supabase

Planned database for storing rooms, members, preference submissions, and votes to keep results fair.

Tables: rooms, members, preferences, matches, votes.

Target browsers

Desktop

Recent Chrome, Firefox, Edge, and Safari (last two versions) at 1280px+.

Mobile

Safari on iOS 16+, Chrome on Android 12+ with responsive layouts under 960px.

Performance goals

Lean assets, CDN-loaded libraries, and static rendering to keep first paint quick on campus Wi-Fi.
